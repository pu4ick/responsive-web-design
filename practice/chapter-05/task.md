### Практическое задание (SaaS Pricing Table)

Мы оставляем задачу с Таблицей тарифов, но усложняем требования к реализации, чтобы задействовать весь арсенал главы.

**Контекст:**
Та же страница тарифов. Но теперь требования к коду жестче. Дизайнер просит "зебру" в списке фич и декоративные галочки.

**Технические требования:**

**1. HTML:**
*   Контейнер `.pricing`.
*   3 карточки `.plan`. Атрибут `data-popular` на средней.
*   Внутри: Заголовок, Цена, Список (`ul > li` — 5-6 пунктов в каждом), Кнопка.
*   *Важно:* Не добавляй классы на `li`. Стилизуй их через селекторы контекста.

**2. CSS (Senior Level):**
1.  **Переменные и HSL:**
    *   Определи основной цвет в HSL в `:root` (например, `--hue: 220; --main-color: hsl(var(--hue), 70%, 50%);`).
    *   Сделай так, чтобы при наведении на карточку, её граница становилась этого цвета, а фон — очень светлой версией этого цвета (меняй только Lightness, например `hsl(var(--hue), 70%, 95%)`).
2.  **Типографика:**
    *   Задай размер заголовка тарифа через `calc()` и `vw`, чтобы он плавно рос.
3.  **Продвинутые селекторы (Список фич):**
    *   Используй `:nth-child(odd/even)`, чтобы покрасить фон нечетных строк списка в легкий серый цвет ("зебра").
    *   Используй `li:not(:last-child)`, чтобы добавить нижнюю границу всем пунктам, кроме последнего.
4.  **Псевдоэлементы:**
    *   Добавь галочку (✔) перед каждым пунктом списка, используя `li::before`. Саму галочку можно вставить символом в `content: "✔ "`.

**Условия:**
*   Минимум классов в HTML. Пусть работает каскад и селекторы.
*   Layout на Grid или Flexbox.

### Дополнительное задание: Виджет "Центр уведомлений"

**Контекст:**
Ты делаешь компонент списка уведомлений для Dashboard.
Уведомления бывают разных типов: **Info** (Инфо), **Success** (Успех), **Warning** (Внимание).
Дизайнер ленивый: он дал только *один* базовый цвет для каждого типа, а тебе нужно сделать красиво (фон посветлее, граница потемнее), используя только CSS-переменные.

**Задача:**
Сверстать список уведомлений, где стили (цвет и иконка) меняются автоматически в зависимости от атрибута `data-type`.

**Технические требования:**

**1. HTML:**
*   Контейнер `.notifications`.
*   4-5 элементов `.alert`.
*   Добавь каждому `.alert` атрибут `data-type`:
    *   `"info"` (обычное)
    *   `"success"` (успех)
    *   `"warning"` (ошибка/предупреждение)
*   Внутри: Заголовок (`h4`), Текст (`p`), Кнопка-крестик "Закрыть" (`button`, можно просто символом `×`).

**2. CSS (Architecture & Selectors):**
*   **HSL & Variables:**
    *   В `:root` задай базовые оттенки (Hue):
        *   `--hue-info: 210;` (синий)
        *   `--hue-success: 120;` (зеленый)
        *   `--hue-warning: 0;` (красный)
    *   Создай *локальную* переменную `--hue` внутри `.alert`, которая по умолчанию равна `--hue-info`.
*   **Attribute Selectors:**
    *   Используй селекторы `[data-type="success"]`, чтобы переопределить локальную переменную `--hue` на `--hue-success`. (Магия CSS: ты меняешь одну цифру, а перекрашивается весь блок).
*   **Theming:**
    *   Фон алерта: `hsl(var(--hue), 80%, 95%)` (очень светлый).
    *   Левая граница (акцент): `hsl(var(--hue), 60%, 50%)` (насыщенный).
    *   Заголовок: `hsl(var(--hue), 80%, 30%)` (темный).
*   **Pseudo-elements (Иконки):**
    *   Используй `::before`, чтобы добавить цветной кружок или символ перед заголовком. Цвет кружка тоже должен зависеть от `--hue`.
*   **Structural Pseudo-classes:**
    *   Используй `:first-child` и `:last-child`, чтобы скруглить углы только у первого и последнего уведомления в списке (сам список должен выглядеть как единый блок).

**Условия:**
*   Не пиши отдельные блоки стилей типа `.alert-success { background: green; }`.
*   Напиши стили **один раз** для `.alert`, используя переменную `--hue`.
*   А в селекторах `[data-type="..."]` меняй **только значение** этой переменной.
