
### 2. Практическое задание (Registration Form)

**Контекст:**
Ты верстаешь форму регистрации для финтех-сервиса. Требования к UX очень высокие.

**Задача:**
Реализовать форму с 3 полями, используя паттерн **Floating Labels** и валидацию на чистом CSS (подсветка ошибок).

**Технические требования:**

**1. HTML:**
*   Контейнер формы.
*   3 поля (каждое в обертке `.form-group`):
    *   **Имя** (`type="text"`).
    *   **Email** (`type="email"`).
    *   **Пароль** (`type="password"`).
*   Кнопка "Создать аккаунт".
*   *Важно:* Используй правильный порядок тегов для CSS-трюка (Input перед Label).

**2. CSS:**
*   **Состояние покоя:** Лейбл серого цвета находится внутри поля (по центру по вертикали).
*   **Состояние фокуса/заполненности:** Лейбл плавно уменьшается (`scale`) и уезжает вверх (`translateY`), меняет цвет на акцентный (синий/фиолетовый).
*   **Валидация:**
    *   Если поле `:invalid` и при этом **не пустое** (`:not(:placeholder-shown)`), покрась границу поля в красный.
    *   *Senior Tip:* Не показывай ошибку, пока пользователь не начал печатать (то есть пока поле пустое, оно не должно быть красным, даже если `required`).

**Условия:**
*   Никакого JS.
*   Используй `transition` для плавности перемещения лейбла.
