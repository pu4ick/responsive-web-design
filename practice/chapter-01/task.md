### 2. Практическое задание (Business Case)
**Контекст:**
Ты работаешь над запуском технического блога компании. Дизайнер еще рисует макеты, но контент-менеджер уже прислал текст статьи. Твоя задача — создать **HTML-каркас (Wireframe)** страницы статьи. CSS пока не трогаем (разве что совсем чуть-чуть для визуализации границ, если хочешь).
**Требования к коду:**
1. Создай файл `index.html`.
2. Используй правильный `doctype` и структуру HTML5.
3. Обязательно добавь метатег `viewport`.
4. Реализуй следующую структуру, используя **только** семантические теги (избегай `div`, где это возможно):
* **Шапка сайта:** Логотип (текстом или картинкой) и Главное меню (Ссылки: "Статьи", "О нас", "Контакты").
* **Основная область (`main`):**
* **Статья (`article`):**
* Заголовок статьи (`h1`).
* Мета-информация (дата публикации, автор).
* Вводный абзац (Lead text).
* Картинка-иллюстрация с грамотным `alt`-текстом.
* Пара разделов с подзаголовками (`h2`).
* **Боковая панель (`aside`):**
* Блок "Об авторе" (краткое био).
* Блок "Похожие статьи" (список ссылок).
* **Подвал (`footer`):** Копирайт и контакты.
**Условия:**
* Не пиши CSS стили для красоты. Нас интересует только структура.
* Код должен быть валидным.
