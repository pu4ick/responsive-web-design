## Урок 5 (Расширенный): Продвинутые селекторы, HSL и Типографика

### 1. Теория: Селекторы-снайперы

Вместо того чтобы вешать класс на каждый элемент, мы используем **комбинаторы** и **псевдоклассы**. Это делает HTML чище.

**A. Комбинаторы (Отношения между элементами)**
*   **Дочерний селектор (`>`)**: `ul > li` — стилизует только прямых детей. Если внутри `li` будет еще один `ul`, его пункты не пострадают.
*   **Соседний селектор (`+`)**: `h2 + p` — стилизует параграф, который идет *сразу* после заголовка (например, делает первую строку жирной).
*   **Селектор последующих элементов (`~`)**: `input:checked ~ .content` — стилизует блок `.content`, если он находится где-то после выбранного чекбокса (на этом строятся CSS-аккордеоны без JS).

**B. Псевдоклассы (Логика в CSS)**
*   **:nth-child(formula)**: Самый мощный инструмент для таблиц и списков.
    *   `:nth-child(odd/even)` — зебра.
    *   `:nth-child(3n)` — каждый третий.
    *   `:nth-last-child(1)` — последний элемент (полезно, чтобы убрать border-bottom).
*   **:not(selector)**: Селектор отрицания.
    *   *Пример:* `li:not(:last-child) { border-bottom: 1px solid #ccc; }` — добавляет линию всем, кроме последнего. Больше не нужно создавать класс `.last-item`.
*   **:empty**: Скрывает пустые контейнеры. `.alert:empty { display: none; }`.

**C. Псевдоэлементы (Декор)**
*   **`::before` / `::after`**: Создают псевдо-теги внутри элемента. Используются для иконок, декоративных линий, кавычек, чтобы не мусорить в HTML пустыми `<span>`.
    *   *Важно:* Всегда требуют свойство `content: ''`.

### 2. Теория: Умные цвета (HSL)

Бен Фрэйн настоятельно рекомендует переходить с HEX (`#333333`) на **HSL** (`hsl(0, 0%, 20%)`).

**Почему это круто:**
*   **H (Hue)**: Оттенок (0-360).
*   **S (Saturation)**: Насыщенность (%).
*   **L (Lightness)**: Светлота (%).

*Бизнес-кейс:* Тебе нужно сделать кнопку и при наведении (hover) сделать её чуть темнее.
*   В HEX: Тебе нужно открывать палитру и подбирать новый код.
*   В HSL: Ты просто уменьшаешь **L** на 10%. `hsl(200, 50%, 50%)` -> `hsl(200, 50%, 40%)`. Это можно делать даже формулами в `calc()`.

### 3. Теория: Резиновая типографика (Fluid Type)

Как сделать так, чтобы шрифт плавно рос от телефона к десктопу, без резких скачков на брейкпоинтах?
Используем `calc()` и единицы вьюпорта (`vw`).

*   *Формула:* `font-size: calc(1rem + 1vw);`
    *   `1rem` — база (чтобы текст не стал микроскопическим на телефоне).
    *   `+ 1vw` — добавка, которая зависит от ширины экрана. Чем шире экран, тем крупнее текст.
