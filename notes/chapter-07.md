Глава 8 книги Бена Фрэйна посвящена **SVG (Scalable Vector Graphics)**.
Для Junior-разработчика SVG — это просто "картинка, которая не пикселит".
Для Senior-разработчика SVG — это **документ внутри документа**, которым можно управлять через CSS (менять цвет, толщину линий, анимировать) и который нужно жестко оптимизировать.

В 2024+ году использовать PNG для иконок — моветон. Мы будем строить **SVG-систему**.

## Урок 7: Векторная графика в продакшене (SVG)

### 1. Теоретический минимум (Senior Notes)

**A. SVG — это XML, а не битовая карта**
JPG/PNG хранят информацию о пикселях. SVG хранит математические формулы (координаты точек, кривые).
*   **Плюс:** Бесконечное масштабирование, малый вес, доступ к внутренностям через CSS/JS.
*   **Минус:** Если в SVG миллион точек (фотореализм), браузер "умрет", пытаясь это отрисовать.

**B. Способы вставки (Critical)**
Это самый важный выбор архитектора.
1.  **`<img>` / `background-image`:** Браузер воспринимает SVG как "глухую" картинку.
    *   *Кэшируется:* Да.
    *   *Стилизуется:* **Нет**. Ты не сможешь поменять цвет иконки при наведении (`:hover`).
2.  **Inline SVG (`<svg>...</svg>` в HTML):** Код графики вставляется прямо в верстку.
    *   *Кэшируется:* Нет (грузится с HTML).
    *   *Стилизуется:* **Да**. Полный контроль через CSS (`fill: red`).
3.  **SVG Sprites (`<symbol>` + `<use>`):** Золотая середина. Мы определяем иконки один раз в начале страницы, а потом переиспользуем их по ID.

**C. CSS для SVG**
Забудь про `background-color` и `border`. У SVG свой словарь:
*   **`fill`**: Цвет заливки.
*   **`stroke`**: Цвет обводки (линии).
*   **`currentColor`**: Ключевое слово в CSS. Если задать `fill: currentColor`, иконка примет цвет текста родителя (`color`). Это киллер-фича для кнопок и ссылок.

**D. Атрибут `viewBox`**
Это "камера", через которую мы смотрим на иконку.
`viewBox="0 0 24 24"` — координаты x, y, ширина, высота. Если убрать этот атрибут, иконка может перестать масштабироваться. Никогда не удаляй его при оптимизации.
