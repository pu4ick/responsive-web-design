## Урок 9: Формы, Валидация и UX

### 1. Теоретический минимум (Senior Notes)

**A. Типы инпутов (Это не только для семантики)**
Когда ты пишешь `type="email"` или `type="tel"`, ты не просто валидируешь данные. Ты **управляешь клавиатурой смартфона**.
*   `type="email"` → появится кнопка `@`.
*   `type="tel"` → откроется цифровая клавиатура (Numpad).
*   `type="search"` → кнопка "Enter" превратится в "Search".
*   *Senior rule:* Всегда используй правильный тип. Заставлять пользователя вводить телефон на QWERTY-клавиатуре — это грех.

**B. Псевдоклассы состояния**
CSS теперь умеет то, для чего раньше нужен был JS:
*   `:focus` — пользователь кликнул в поле.
*   `:valid` / `:invalid` — данные соответствуют (или нет) атрибутам `type` или `pattern`.
*   `:placeholder-shown` — **Киллер-фича**. Позволяет определить, пустой инпут или нет (если плейсхолдер показан — значит, поле пустое).
*   `:focus-within` — если фокус находится внутри этого контейнера (очень полезно для стилизации обертки).

**C. Трюк "Floating Label" (Плавающая подпись)**
Чтобы сделать лейбл, который живет внутри поля, а потом улетает, нужно:
1.  Поставить `<label>` **ПОСЛЕ** `<input>` (чтобы использовать селектор соседа `+`).
2.  Дать инпуту `placeholder=" "`.
3.  Использовать связку `input:not(:placeholder-shown) + label`.
