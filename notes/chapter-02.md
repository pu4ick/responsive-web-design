## Урок 2: Логика адаптивности и Mobile-First

### 1. Теоретический минимум (Senior Notes)

**A. Метатег Viewport (Повторение — мать учения)**
В прошлом уроке ты добавил `<meta name="viewport" content="width=device-width, initial-scale=1.0">`.
*   **Зачем:** Без него мобильный браузер "соврет" CSS-у, что ширина экрана — 980px (как у старого десктопа), и просто уменьшит масштаб страницы до нечитаемого состояния.
*   **device-width**: Говорит браузеру: "Считай шириной страницы реальную ширину экрана в аппаратно-независимых пикселях".

**B. Синтаксис Media Queries**
Медиазапрос — это условие. "ЕСЛИ ширина экрана такая-то, ТО примени эти стили".

```css
@media (min-width: 768px) {
  /* Стили для планшетов и десктопов */
}
```

**C. Стратегия Mobile-First (Критически важно)**
Существует два подхода к верстке:

1.  **Desktop-First (Устаревший):** Ты верстаешь сайт для монитора. Потом пишешь `@media (max-width: 768px)`, чтобы *отменить* или *переписать* стили для телефона.
    *   *Проблема:* Ты пишешь код, чтобы потом его "сломать". Браузер телефона загружает сложные десктопные стили, парсит их, а потом переписывает мобильными. Это медленно.

2.  **Mobile-First (Индустриальный стандарт):**
    *   Сначала ты пишешь CSS для **мобильного телефона** (без всяких медиазапросов). Это твой `base.css`. Обычно это одна колонка, простой поток.
    *   Потом ты добавляешь `@media (min-width: ...)`, чтобы **добавить** сложность для больших экранов (поставить колонки рядом, увеличить шрифты).
    *   *Плюс:* Браузер телефона видит только базовые стили и игнорирует блок внутри медиазапроса. Это быстро и логично (от простого к сложному).
